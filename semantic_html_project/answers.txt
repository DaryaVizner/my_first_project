1. Чем семантическая верстка отличается от несемантической и почему она важна?
 - Семантическая верстка использует теги, которые несут смысловую нагрузку (<article>, <section>, <nav>), а несемантическая - только <div> и <span>. Она важна для доступности, SEO и поддержки кода.

2. В каких случаях следует использовать <article> вместо <section>? Приведите примеры из вашей работы.
 - <article> - для самостоятельного контента (статьи, новости), <section> - для тематической группировки. В моей работе: каждый факт о енотах - это <article>, так как это независимый контент.

3. Для чего предназначены теги <header> и <footer> и могут ли они использоваться несколько раз на странице?
 - <header> - вводная информация, <footer> - заключительная. Да, могут использоваться несколько раз - для всей страницы и для каждого <article>.

4. Какой тег следует использовать для основного контента страницы и почему не стоит использовать для этого <div>?
 - <main> - для основного контента. <div> не несет смысла, а <main> помогает скринридерам и поисковикам.

5. В чем разница между <nav> и обычным списком ссылок? Когда обязательно использовать <nav>?
 - <nav> - для основной навигации, список - просто ссылки. <nav> обязательно для главного меню.

6. Для чего используется атрибут datetime в теге <time> и почему он важен?
 - Указывает машинно-читаемую дату. Важен для поисковиков и accessibility.

7. Какие преимущества дает использование <figure> и <figcaption> вместо простого <img> с подписью?
 - Семантически связывают изображение с подписью, улучшая доступность и SEO.

8. Когда следует использовать <aside> и какой контент в него помещать?
 - Для дополнительного контента (сайдбар, примечания). В моей работе - блок "Популярные факты".

9. Как семантическая разметка влияет на SEO и доступность веб-страницы?
 - Улучшает индексацию поисковиками и работу скринридеров для слабовидящих.

10. Какие теги вы использовали для улучшения структуры документа и как они помогают пользователям и поисковым системам?
<main> - основной контент
<article> - независимые статьи
<section> - тематические разделы
<nav> - навигация
<aside> - дополнительная информация
<figure> + <figcaption> - изображения с подписями
<header>/<footer> - шапка и подвал
